# 中文写作规范校验

对中文文章进行全面的写作规范检查，输出结构化的校验报告和修改建议。

## 使用方式

用户提供文章内容（文本、文件路径或 URL），按以下七大类逐项检查。

## 校验流程

### 第一步：获取文章内容

- 如果用户给了文件路径，读取文件
- 如果用户给了 URL，使用 agent-fetch 抓取
- 如果用户直接粘贴了文本，直接使用

### 第二步：逐项检查

按以下 7 个类别检查，每个类别内逐条扫描。仅报告发现问题的条目——无问题的类别跳过不报告。

### 第三步：输出报告并提供修改

使用以下格式输出（仅包含有问题的类别）：

```markdown
# 📝 中文写作规范校验报告

## 总评
- 检查条数：X
- 发现问题：X 处
- 严重程度分布：🔴 X / 🟡 X / 🟢 X

---

## 🔴 [类别名] — 问题 N 处

### 问题 1
- **位置**：第 X 段 / 原文 `...`
- **问题**：具体描述
- **修改建议**：`修改后的文本`

---

## ✅ 总结与建议
[给出 2-3 条最值得优先修改的建议]
```

### 第四步：应用修改

报告输出后，询问用户是否需要直接修改文章。如果用户确认：

1. 将所有 🔴 错误项自动修正（标点、空格、大小写等机械性问题可直接改）
2. 将 🟡 建议项标注修改方案，由用户确认后再改
3. 🟢 优化项仅供参考，不自动修改
4. 输出修改后的完整文章，并在末尾附上修改摘要（共改了几处、改了什么）

严重程度：
- 🔴 **错误**：标点错误、禁用词、专有名词大小写错误、空格缺失
- 🟡 **建议**：欧化表达、慎用词、陈词滥调、文风问题
- 🟢 **优化**：可进一步精简、更自然的表达方式

## 七大检查类别

### 一、标点符号

**中文标点：**
- [ ] 全角标点（`，。！？；：` 而非半角）
- [ ] 直角引号（`「」` `『』` 而非 `""` `''`）
- [ ] 破折号 `——`（两个 em dash，而非 `--` 或单个 `—`）
- [ ] 省略号 `……`（而非 `。。。` 或六个点 `......`）
- [ ] 斜杠用 `/` 不用 `\`
- [ ] 不重复使用标点符号（`！` 而非 `！！！`）
- [ ] 全角标点与其他字符之间不加空格（`iPhone，好开心` 而非 `iPhone ，好开心`）

**英文标点：**
- [ ] 半角标点
- [ ] 标点后留一个空格
- [ ] 省略号 `…`（单个符号，而非三个点）
- [ ] 破折号 `—`（em dash）、范围 `–`（en dash）
- [ ] 括号外有空格、内无空格
- [ ] 完整英文句子内使用半角标点（`Stay hungry, stay foolish.` 而非 `Stay hungry，stay foolish。`）
- [ ] 数字使用半角字符（`1000` 而非 `１０００`）

### 二、中英文混排

- [ ] 中英文之间有空格
- [ ] 中文与数字之间有空格
- [ ] 数字与单位之间有空格（`10 Gbps` 而非 `10Gbps`），但度数/百分比除外（`90°`、`15%`）
- [ ] 专有名词大小写正确（GitHub / iPhone / JavaScript / macOS / iOS / Wi-Fi 等）
- [ ] 不使用不地道的缩写（`TypeScript` 而非 `Ts`，`HTML5` 而非 `h5`）
- [ ] 产品名按官方格式书写（如 `豆瓣FM`、`LeanCloud`）

### 三、数字用法

- [ ] 一位数使用汉字（`三种` 而非 `3 种`）
- [ ] 两位以上使用阿拉伯数字（`365 天` 而非 `三百六十五天`）
- [ ] 阿拉伯数字前后有空格

### 四、用词规范

- [ ] 无禁用词汇（屌丝、白富美、高富帅、高大上、富二代、安利（动词）、撕逼、尼玛、也是醉了、有木有、美女/帅哥（称呼）、给力）
- [ ] 慎用词汇是否恰当（情怀、物欲横流、众所周知、简约、唯美、精品、行为艺术、Low、人工智能）
- [ ] 无陈词滥调或空洞套话
- [ ] 无网络热词/流行语入侵正式写作
- [ ] 专有名词含义准确（如"伪命题"是否被误用）
- [ ] 粗俗词的柔化变体同样不可接受（如「关我 P 事」）

### 五、避免欧化中文

- [ ] 无多余量词/代词（不必要的「一个」「一种」「这些」「那个」）
- [ ] 无抽象名词泛滥（「对问题的解决」→「解决问题」）
- [ ] 无不必要的中英夹杂（能用中文说清楚的不夹英文）
- [ ] 无不必要的音译（有现成中文词汇时不用音译）

### 六、文风检查

- [ ] 无新闻联播体（`带着这个问题，记者走访了……`）
- [ ] 无淘宝体（`亲，请给我们点个赞哦`）
- [ ] 无自嗨型写作（只关注自身不考虑读者）
- [ ] 无耍小聪明的表述（为炫技而非为沟通）
- [ ] 表达清晰直接，说人话

### 七、翻译/双语

（仅当文章含中英双语内容时检查）
- [ ] 中英文语义对应准确
- [ ] 英文部分无明显语法/拼写错误

## 注意事项

- 创作类文章（小说、诗歌等）的文风检查应放宽，侧重标点和用词
- 技术文档中的英文术语保留是合理的，不按"中英夹杂"报错
- 直接引语中出现禁用词不报错
- 如果文章整体质量很好，也要给出正面反馈
